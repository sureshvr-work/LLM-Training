Agent-to-agent · A2A

Trip Planner

Watch the whole handshake: the planner discovers four agents, turns each card's example into a real A2A message, then the agents run their own SENSE · REASON · ACT loops. An A2A message has no skillId — the card's examples are the only bridge from "what I want" to "what to say".

1
Discovery — GET each Agent Card
REST GET /.well-known/agent-card.json. The card advertises the agent's skill and example phrasings.
2
Compose the A2A message — the example is the bridge
The client can only send text (no skillId). How should it phrase that text from the card's example?
3
The agents work — SENSE · REASON · ACT
Each agent receives the message over JSON-RPC and runs its own loop. It never sees A2A again.
