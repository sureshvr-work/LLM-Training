The fix · one shared contract

A2A is a horizontal bus every agent speaks.

Define the contract once; every agent implements the same one. Any agent can discover, ask, and delegate to any other — across vendors, frameworks and clouds.
What's shared vs. what stays private
Advertised — the agent's skills & capabilities, so others know what they can ask it to do.
Private — its internal state, memory & tool implementations: how it does the work.
You invoke a skill it advertised — never its internal tools. (State/memory privacy applies to any framework; A2A adds the standard way to advertise and call skills.)
● core — A2A, one shared contract
● ring 1 — the operations (same for all)
● ring 2 — communication: JSON-RPC · gRPC · REST
● spokes — each agent picks a protocol; contract identical
N × M → N + M. Same contract inside · your choice of protocol on the way out.
▶ Watch: the Service agent sends an A2A message to Research — same contract, each on its own protocol.
Orientation · A2A vs MCP

A2A is the horizontal bus. MCP is the vertical one.

A2A · across the top. Google's auto-repair-shop: the specialists talk agent-to-agent — Diagnostics hands findings to Estimate, Estimate asks Parts. Each stays a black box; only skills are shared.
MCP · dropping down. Each agent reaches its own tools & data over MCP — its private DB, API, inventory. Vertical, and never exposed to the other agents.
Same loop inside each: SENSE→REASON→ACT. ACT is where both happen — call a tool (MCP) or delegate to a peer (A2A). Complementary layers — you'll use both.
Two roles: a Client Agent formulates the task, a Remote Agent executes it.

A2A defines the operations once, then binds them to three transports. JSON-RPC is the canonical one — that's what we'll use. To the client, the remote agent is a black box reached over HTTP.
AgentCard
discovery
JSON at /.well-known/agent-card.json — identity, endpoint url, capabilities, skills, auth. How you find an agent and learn what it does.
Task
the unit of work
Server-created, stateful: id, contextId, status.state, artifacts[], history[]. Everything revolves around it.
TaskState
lifecycle
submitted → working → input-required → completed. Terminal: completed, canceled, failed, rejected.
Message · Part
a turn
A Message has a role (user/agent) and parts[]. A Part is text, file, or data (structured JSON).
Artifact
the output
What the task produces — artifactId, name, parts[]. Can stream in incrementally as the task runs.
contextId · taskId
two ids that thread it
contextId groups a conversation (a thread); taskId is one request within it. Omit on first call; the server mints them.
