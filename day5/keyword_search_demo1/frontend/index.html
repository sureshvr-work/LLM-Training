Keyword Lab — watch every stage of the inverted index

One document, end to end: upload → chunk → build the inverted index → search (BM25) → answer. The sparse counterpart to the dense RAG Lab.

No embedding model. No vectors. Keyword search analyzes text into terms, files each term under the chunks that contain it, and ranks by exact-term overlap (BM25). The same chunk you would embed, you also index — so step 2 lets you pick any chunking strategy that applies to an inverted index.

3

Pick a chunking strategy

Six of the seven taxonomy families produce chunks an inverted index can hold. (Embedding-time is vector-only — see the note.)

4

Build the inverted index

The analyzer turns each chunk into terms; every term points to the chunks that contain it. This is the whole data structure BM25 reads.

5

Ask & retrieve (BM25)

Your query runs through the same analyzer as the index — then BM25 scores every chunk by term overlap.

6

Send to the LLM

The retrieved chunks become the context. (Hierarchical returns the parent chunk for fuller context.)
