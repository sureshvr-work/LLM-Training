Part 1 — Chunking Strategy by Use Case

Chunking Strategies

Fixed-size + overlap

E-commerce / Customer Support

Product FAQ Chatbot

Thousands of short product descriptions and FAQ entries. No consistent structure — just flat text.

chunk_size = 256 tokens overlap = 30 tokens → embed → vector DB → user asks "does it fit 13 inch laptop?" → retrieve top-3 → send to LLM
simple setup high volume short queries
Paragraph-based

Legal / Finance

Contract Review Assistant

Legal contracts have clear paragraph structure. Each paragraph covers one clause — splitting mid-paragraph loses meaning.

split at blank lines / paragraph breaks → token-cap each paragraph at 512 → embed → store with metadata: {clause_type, page, contract_id} → query: "termination clause conditions?"
structure-aware metadata filtering clause search
Document-structure

Healthcare

Clinical Guidelines RAG

Medical PDFs have sections: Diagnosis, Treatment, Dosage, Contraindications. Split by heading — each section is one chunk.

parse headings → split by H1/H2/H3 → embed each section → metadata: {section, condition, version} → query: "dosage for pediatric patients?" → filter: section="Dosage"
heading-aware filterable high accuracy
Semantic chunking

Research / Knowledge Base

Scientific Paper Q&A

Research papers shift topics mid-paragraph. Semantic chunking detects topic boundaries using embeddings — chunks align with ideas, not lines.

embed each sentence → compute similarity between adjacent sentences → start new chunk when similarity drops below 0.6 → result: chunks = coherent topics
most accurate slow to build best retrieval
Recursive split

General SaaS

Internal Knowledge Base

Mixed content — Confluence pages, Notion docs, Markdown READMEs. No consistent format. Recursive split handles all gracefully.

try split by: \n\n → \n → . → " " → stop when chunk ≤ 512 tokens → good default for unknown formats
mixed formats safe default LangChain default
Page-level

Insurance / Banking

Policy Document Search

Users ask "what does page 12 say about flood coverage?" One page = one chunk. Simple, auditable, maps directly to physical document.

split by PDF page boundary → token-cap: if page > 1024 tokens, split into top/bottom halves → metadata: {page_number, doc_id} → cite source: "See page 12"
auditable citable compliance-friendly

Part 2 — NLP Model Strategy by Use Case

NLP Model Types in Production

NER

HR / Recruiting

Resume Parser

Extract structured data from unstructured resumes at scale — names, companies, dates, skills — without sending to LLM.

spaCy NER → extract: PERSON → candidate name ORG → employer names DATE → employment dates GPE → locations → store in DB with filters → recruiter searches: "Java dev, 5yr exp, Bangalore"
spaCy local structured output
Sentiment Analysis

Retail / E-commerce

Product Review Dashboard

Classify 100k product reviews per day as positive/negative/neutral. Flag negative spikes for immediate ops alert.

review text → sentiment model (BERT fine-tuned) → score: Negative 0.91 → if score > 0.85 AND volume spike: → alert ops team via Slack → dashboard: sentiment trend by SKU
real-time high volume alerting
Text Classification

Customer Support SaaS

Support Ticket Router

Automatically route incoming support tickets to the right team — Billing, Technical, Account, General — without human triage.

ticket text → fine-tuned classifier → category: "Billing" (0.94 confidence) → route to billing queue → if confidence < 0.6: route to human triage
zero human triage fast fallback to human
Text Similarity

Legal Tech

Duplicate Case Detector

Law firms file thousands of cases. Detect near-duplicate cases to avoid rework — "this case is 91% similar to case #4521."

new case filing → embed with text-embedding-3-small → cosine similarity vs all past cases → if similarity > 0.90: → flag as potential duplicate → show top-3 similar cases to lawyer
embeddings cosine similarity deduplication
Summarization

Finance / Banking

Earnings Call Digest

Analysts read 50-page earnings call transcripts. Auto-summarize into 5 bullet points — revenue, guidance, risks, highlights.

earnings transcript (50 pages) → chunk by paragraph → extractive summary per section → LLM abstractive summary of summaries → output: 5-bullet digest email → sent to analysts before market open
two-stage batch job cost-efficient
PII Redaction (Presidio)

Healthcare / Finance

HIPAA-Compliant LLM Queries

Hospital wants to use LLM to summarize patient notes. PII must never leave the org. Presidio redacts locally before sending.

patient note (raw) → Presidio (local, no internet) → "John Doe, SSN 123-45" → "[PERSON], SSN [US_SSN]" → redacted text → LLM API → LLM response → return to doctor (PII never left the hospital network)
local only HIPAA compliance
Relation Extraction

Supply Chain / Enterprise

Knowledge Graph Builder

Build a graph of supplier relationships from 10 years of procurement documents automatically.

"Acme Corp supplies steel to Ford" → relation extraction model → (Acme Corp) --SUPPLIES--> (Ford) → entity: steel, category: raw_material → store in graph DB (Neo4j) → query: "who are Tier-2 suppliers for Ford?"
graph DB supply chain structured data
Topic Modeling

Media / Publishing

News Article Organizer

Auto-tag 10,000 articles per day into topics without manual labelling. Feed into recommendation engine.

article text → LDA / BERTopic model → Topic 1: {economy, inflation, RBI} → Finance → Topic 2: {match, wicket, IPL} → Cricket → tag stored with article → recommendation: show Finance articles to users who read Finance
unsupervised auto-tagging recommendation
