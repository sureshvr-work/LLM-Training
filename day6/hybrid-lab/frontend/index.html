Upload → chunk once → build embeddings and an inverted index from the same chunks → search with cosine, BM25, and an RRF fusion → answer.

Dense and sparse, side by side. The dense lane matches by meaning (vector cosine); the sparse lane matches by exact terms (BM25). The hybrid lane fuses both rankings with Reciprocal Rank Fusion so a chunk that either lane likes rises to the top.
