Vectorless RAG Lab — BM25 keyword retrieval

Same pipeline as vector RAG, but retrieval uses keyword scoring instead of embeddings. Watch every stage.

This demo — Vectorless RAG (BM25)
  • No embedding model — zero API cost for indexing
  • Index builds in milliseconds
  • Finds chunks that share exact keywords with your query
  • Misses synonyms: "heart attack" ≠ "myocardial infarction"
  • Great for exact terminology (legal, medical, code)
Vector RAG (rag_demo1)
  • Embedding API call for every chunk + every query
  • Index takes seconds to minutes depending on size
  • Finds semantically similar text even without shared words
  • "forgot credentials" finds "password reset" content
  • Better for conversational / paraphrase-heavy queries

Choose the LLM model

3

Chunk the document

Fixed-size character windows with overlap. Overlap highlighted in yellow — it prevents answers from being cut at chunk boundaries.

4

Build keyword index (BM25)

Tokenise every chunk, remove stop words, count term frequencies, compute IDF weights. No API call. Runs in milliseconds on your machine.

5

Search with BM25

Your query is tokenised (stop words removed). Each chunk gets a BM25 score based on how many query terms it contains and how rare those terms are. Yellow highlight = matched keyword.

6

Send to the LLM

The top-K BM25 chunks become the context. The LLM answers from those chunks only and cites each passage.
